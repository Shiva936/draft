# Brand Guidelines

These guidelines help keep public references to Draft clear and consistent. They are project style guidance, not a trademark policy.

## Name

- Use `Draft` for the project.
- Use `draft` for the CLI command.
- Use “Draft project” or “Draft contributors” when referring to the open-source community.

## Description

Preferred short description:

> Draft is a local-first review, verification, approval, receipts, and rollback layer for software changes.

Avoid saying Draft replaces Git, code hosts, CI, deployment systems, agents, or editors. Draft is designed to sit in front of those tools as a local review boundary.

## Logos And Assets

Project assets live under `assets/`. Use them in ways that make it clear you are referring to this project. Do not imply endorsement, certification, or official hosted service status unless that relationship exists.

## Tone

Keep public language precise, calm, and implementation-grounded. Prefer “local,” “reviewable,” “receipt-backed,” and “auditable” over broad claims.
